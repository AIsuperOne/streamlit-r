# 5G基站扇形覆盖图 — 技术文档

## 1. 项目概述

本项目是一个基于 Streamlit + 高德地图3D的 5G 基站覆盖可视化工具，用于展示咸宁地区 5G 基站的扇形覆盖、采样点测量数据（RSRP/SINR/覆盖率/波束）、栅格覆盖率分析和覆盖比例分析。数据来源于本地 SQLite 数据库（2595 万行 MR 采样数据 + 5.3 万基站小区数据）。

### 技术栈

- **后端**：Python 3 + SQLite + Shapely（地理空间计算）
- **前端**：Streamlit + 高德地图 JS API 2.0（3D视角，pitch=50°）
- **通信**：Streamlit 内嵌 HTTP API 服务器（端口 8503），JS 通过 fetch 调用

### 代码架构（app.py ~1750行）

```
app.py
├── 1. 常量与配置          ~56行
├── 2. 算法层              ~180行  纯函数，无副作用，不访问数据库
├── 3. 数据层              ~260行  数据库访问函数
├── 4. API层               ~120行  HTTP路由 + 瘦handler
├── 5. 呈现层              ~1130行  HTML/JS模板 + Streamlit入口
```

## 2. 功能描述

### 2.1 基站扇形覆盖图

启动后自动加载 5GBaseStation 表中所有有效基站，在每个小区的 RRU 位置绘制扇形覆盖区域：

- **扇形角度**：以方位角为中心，左右各扩展 60°（共 120°）
- **扇形半径**：默认 50 米，可选 100/150/自定义（1-5000 米）
- **颜色区分**：按工作频段着色（700MHz 金色，其余蓝色）
- **3D视角**：默认3D模式，pitch=50°俯视角，支持鼠标旋转/倾斜
- **交互**：点击扇形选中/取消小区，选中后高亮为绿色
- **方位角显示**：已选中小区列表显示方位角信息

### 2.2 散点测量模式（RSRP/SINR/波束/覆盖率）

选中小区后，默认进入散点模式，从 API 获取该小区最多 5000 个采样点，以 CircleMarker 渲染在地图上：

- **RSRP 模式**：按 RSRP 值分 4 级着色（优 ≥-95dBm 绿 / 中 -105~-95 黄 / 差 -115~-105 橙 / 极差 <-115 红）
- **SINR 模式**：按 SINR 值分 4 级着色（优 ≥15dB 绿 / 中 5~15 黄 / 差 -3~5 橙 / 极差 <-3 红）
- **波束模式**：以基站方位角为参考，±60° 范围内为波束内（绿色），范围外为波束外（红色），统计波束内/外数量及占比。同时计算并显示最优方位角（按当前室内外筛选，0°~355°每5°扫描波束内占比最大的角度）
- **覆盖率模式**：判定每个采样点是否满足 RSRP>-105 且 SINR>-3，覆盖点绿色/未覆盖点红色，统计覆盖/未覆盖数量及覆盖率百分比
- **室内/室外过滤**：下拉选择 全部/室内/室外

### 2.3 栅格分析模式

切换到栅格模式后，将小区采样点区域划分为 5m×5m 正则栅格，支持两个子模式：

#### 2.3.1 小区占比

- **cell_count**：属于该小区的采样点数
- **plmn_count**：属于 PLMN=46000 的所有采样点数
- **覆盖率**：cell_count / plmn_count × 100%
- **着色**：按覆盖率分 4 级（低覆盖 0~20% 红 / 中覆盖 20~50% 黄 / 中高覆盖 50~80% 蓝 / 高覆盖 80~100% 绿）

#### 2.3.2 覆盖比例

- **weak_count**：栅格内 rsrp<-105 或 sinr<-3 的采样点数
- **weak_pct**：weak_count / plmn_count × 100%（弱覆盖占比）
- **着色**：按弱覆盖占比分 4 级（覆盖优良 0~20% 绿 / 覆盖良好 20~50% 黄 / 覆盖稍弱 50~80% 橙 / 覆盖很弱 80~100% 红）
- **覆盖面积**：栅格数 × 25m²

### 2.4 采样点详情

在散点模式下，鼠标悬停在采样点上 2 秒后，弹出 InfoWindow 显示：

- GNBID / CI / 主服务频点 / RSRP / SINR / 邻区 RSRP 列表 / 邻区 PCI 列表 / 邻区频点列表

### 2.5 栅格详情

在栅格模式下，鼠标悬停在栅格上 2 秒后，弹出 InfoWindow 显示：

- 栅格小区采样点数 / 总采样点数 / 覆盖率
- 按小区分组的详细统计：NR频点 / GNBID / CI / RSRP均值 / SINR均值 / 采样点数

## 3. 应用算法

### 3.1 坐标转换算法 — `meters_to_degrees`

**功能**：将米制距离转换为经纬度偏移量

**实现**：
```
经度偏移 = meters / 111320.0 / cos(avg_lat × π / 180)
纬度偏移 = meters / 110540.0
```

**原理**：地球近似椭球体，1°纬度 ≈ 110540m 基本恒定；1°经度 ≈ 111320m × cos(纬度)，纬度越高经度跨度越短。5m 对应约 0.00005°。

**输入**：meters(米), avg_lat(参考纬度)
**输出**：(lng_offset, lat_offset)

### 3.2 栅格原点算法 — `calc_grid_origin`

**功能**：确定栅格坐标系的随机原点

**实现**：
```
random.seed(f"{gnbid}_{ci}")         # 确定性种子
ref_idx = random.randint(0, N-1)     # 随机选一个采样点
origin_lng = pts[ref_idx].lng - cell_lng / 2
origin_lat = pts[ref_idx].lat - cell_lat / 2
```

**原理**：用 gnbid_ci 作为随机种子，保证同一小区多次计算生成相同的栅格原点。偏移半格使参考点位于栅格中央而非边界。

**输入**：cell_pts, cell_lng, cell_lat, gnbid, ci
**输出**：(origin_lng, origin_lat)

### 3.3 数学落格算法 — `bin_point_to_grid`

**功能**：将采样点经纬度坐标映射到栅格索引

**实现**：
```
gx = math.floor((lng - origin_lng) / cell_lng)
gy = math.floor((lat - origin_lat) / cell_lat)
```

**关键**：用 `math.floor()` 而非 `int()`。`int(-8.9) = -8`（截断向零），`floor(-8.9) = -9`（向负无穷），后者才是正确的格索引。

**输入**：lng, lat, origin_lng, origin_lat, cell_lng, cell_lat
**输出**：(gx, gy) 整数栅格索引

### 3.4 栅格构建算法 — `build_grid_polygons`

**功能**：将小区采样点落格并构建 Shapely Polygon 列表

**实现**：
1. 遍历每个采样点，执行数学落格，统计 `(gx, gy)` 格子的 cell_count
2. 对每个非空格子，用 `origin + gx * cell_size` 计算 sw/ne 坐标
3. 构建 Shapely `Polygon([(sw_lng, sw_lat), (ne_lng, sw_lat), (ne_lng, ne_lat), (sw_lng, ne_lat)])`

**输入**：cell_pts, origin_lng, origin_lat, cell_lng, cell_lat
**输出**：`[{gx, gy, polygon, cell_count, plmn_count, weak_count}, ...]`

### 3.5 PLMN 统计算法 — `count_plmn_in_polygons`

**功能**：统计每个栅格内的 PLMN 采样点数和弱覆盖点数

**实现（两步过滤）**：
1. **数学落格**（O(1)）：对每个 PLMN 采样点计算 `(gx, gy)`，在 `poly_map` 字典中查找对应栅格
2. **Shapely covers()**（精确）：对候选栅格执行 `polygon.covers(Point(lng, lat))` 精确判定，排除边界点
3. **弱覆盖判定**：落入栅格后判断 `rsrp < -105 or sinr < -3`，累加 `weak_count`

**为什么两步**：数学落格 O(1) 快速定位候选栅格（跳过 99% 不相关栅格），covers() 精确排除落在栅格边界线上的点。

**输入**：plmn_rows[(lng, lat, rsrp, sinr), ...], polygons, origin, cell_size
**输出**：直接修改 `polygons[i]["plmn_count"]` 和 `polygons[i]["weak_count"]`

### 3.6 覆盖率算法 — `calc_coverage`

**功能**：计算单个栅格的小区占比覆盖率

**实现**：
```
pct = round(cell_count / plmn_count × 100)    if plmn_count > 0
pct = 0                                         if plmn_count == 0
```

**含义**：cell_count = 属于该小区的采样点数，plmn_count = 该栅格内 PLMN=46000 的所有运营商采样点数。比值越高，说明该小区在该区域覆盖占比越大。

### 3.7 覆盖比例算法

**功能**：计算单个栅格的弱覆盖占比

**实现**：
```
weak_pct = round(weak_count / plmn_count × 100)    if plmn_count > 0
weak_pct = 0                                         if plmn_count == 0
```

**含义**：weak_count = 栅格内 rsrp<-105 或 sinr<-3 的采样点数，plmn_count = 该栅格内所有 PLMN 采样点数。弱覆盖占比越高，说明该区域信号质量越差。

**覆盖面积**：栅格数 × 25m²（每个栅格 5m × 5m = 25m²）

### 3.8 栅格重建算法 — `reconstruct_polygon`

**功能**：从 grid_params 重建 Shapely Polygon

**实现**：与 `build_grid_polygons` 相同的算术路径：
```
sw_lng = origin_lng + gx × cell_lng
sw_lat = origin_lat + gy × cell_lat
ne_lng = origin_lng + (gx + 1) × cell_lng
ne_lat = origin_lat + (gy + 1) × cell_lat
```

**为什么需要**：前端通过 URL 传递 `origin_lng, origin_lat, cell_lng, cell_lat, gx, gy` 而非 `sw/ne` 坐标，避免浮点数经 URL 传递后的精度丢失（6 位有效数字的经纬度差异可导致栅格偏移）。

### 3.9 Haversine 方位角算法 — `calc_bearing`（Python） / `calcBearing`（JS）

**功能**：计算从基站到采样点的地理方位角

**实现**：
```
d_lng = (lng2 - lng1) × π / 180
y = sin(d_lng) × cos(lat2)
x = cos(lat1) × sin(lat2) - sin(lat1) × cos(lat2) × cos(d_lng)
bearing = (atan2(y, x) × 180 / π + 360) % 360
```

**原理**：Haversine 公式变体，计算大圆航线的初始方位角。结果归一化到 [0°, 360°)。Python版和JS版逻辑完全一致。

### 3.10 最优方位角扫描算法 — `query_optimal_azimuth`

**功能**：扫描所有角度方向，找出波束内采样点占比最大的方位角

**实现（区间覆盖法，O(N)）**：
```python
angle_counts = [0] * 72  # 0°, 5°, 10°, ..., 355°
for b in bearings:
    center = int(round(b / 5)) % 72
    for offset in range(-6, 7):  # ±30°/5 = ±6
        angle_counts[(center + offset) % 72] += 1

best_idx = max(range(72), key=lambda i: angle_counts[i])
best_angle = best_idx * 5
```

**原理**：对每个采样点的方位角b，波束宽度±30°内的所有角度索引都应+1。等价于对每个bearing，将其影响区间[b-30°, b+30°]内的13个角度索引（±6步×5°）各+1。复杂度从O(72×N)优化为O(13×N)，实际提升约5倍。

**室内外筛选**：按当前筛选条件（全部/室内/室外）计算，前端切换筛选后自动刷新。

### 3.11 覆盖率判定算法 — `query_measurements(coverage)`

**功能**：判定每个采样点的覆盖状态

**实现**：
```
覆盖条件：rsrp > -105 AND sinr > -3
covered_flag = 1 if (rsrp > -105 and sinr > -3) else 0
覆盖率 = SUM(covered_flag=1) / COUNT(*) × 100%
```

**分母定义**：有效RSRP+SINR的采样点（rsrp IS NOT NULL AND sinr IS NOT NULL），只有两个值都有效才能判定覆盖/未覆盖。

### 3.12 波束判定算法 — `isInBeam`（JS端）

**功能**：判断采样点是否在基站波束覆盖范围内

**实现**：
```javascript
diff = ((bearing - azimuth + 180 + 360) % 360) - 180   // 归一化到 [-180, 180]
return Math.abs(diff) <= 60
```

**原理**：计算方位角与基站方位角的差值（处理 0°/360° 跨越），若偏差 ≤ 60° 则认为在波束内。

### 3.13 扇形绘制算法 — `createSectorPath`（JS端）

**功能**：生成基站扇形覆盖区域的路径坐标

**实现**：
```javascript
dLng = radius / 111320 / cos(lat)        // 经度方向半径
dLat = radius / 110540                    // 纬度方向半径
for a in (azimuth-60) to (azimuth+60) step 2°:
    point = (lng + dLng × sin(a), lat + dLat × cos(a))
path = [基站位置, ...弧线点, 基站位置]
```

**原理**：以基站为圆心，将半径转换为经纬度偏移后，沿弧线每隔 2° 生成路径点（共 61 个点），构成扇形多边形。

### 3.14 竞态保护算法 — `scatterLoadSeq / gridLoadSeq`（JS端）

**功能**：防止小区取消/重选后旧请求回调仍渲染数据

**实现**：
```javascript
// 取消小区时递增序号
scatterLoadSeq[name] = (scatterLoadSeq[name] || 0) + 1;

// 发起请求前记录当前序号
var mySeq = scatterLoadSeq[name];

// 回调中检查序号是否一致
if (mySeq !== scatterLoadSeq[name]) return;   // 序号已变，丢弃旧结果
```

**原理**：每次取消小区时递增序号，新请求回调中比对自己的序号与当前序号，不一致则说明用户已重新操作，丢弃旧结果。

### 4.4 新增优化列（REAL 类型）

为避免 SQLite 对 TEXT 列的 `CAST()` 导致全表扫描，新增 4 列数值类型：

| 列名 | 类型 | 来源 |
|------|------|------|
| lng | REAL | `CAST(longitude AS REAL)`，空值时为 NULL |
| lat | REAL | `CAST(latitude AS REAL)`，空值时为 NULL |
| rsrp | REAL | `CASE WHEN nr_ssb_rsrp!="" AND nr_ssb_rsrp!="0" THEN CAST(nr_ssb_rsrp AS REAL) ELSE NULL END` |
| sinr | REAL | `CASE WHEN nr_ssb_sinr!="" AND nr_ssb_sinr!="0" THEN CAST(nr_ssb_sinr AS REAL) ELSE NULL END` |

| 索引名 | 定义 | 用途 |
|--------|------|------|
| idx_data_gnbid_ci | `data(gnbid, ci)` | 小区维度查询（measurements, cell_points） |
| idx_data_gnbid_ci_plmn | `data(gnbid, ci, plmn)` | 小区+PLMN 查询（cell_points 带信号过滤） |
| idx_data_plmn_earfcn_lng_lat_rsrp_sinr | `data(plmn, nr_earfcn, lng, lat, rsrp, sinr)` | PLMN 空间查询覆盖索引（grid_analysis 性能关键，含sinr避免回表） |
| idx_bs_gnbid_cellid | `"5GBaseStation"("Gnbid", "Cellid")` | 基站频点/坐标查询 |

## 5. 呈现实现

### 5.1 技术架构

```
┌──────────────┐     HTTP/JSON      ┌──────────────┐     SQL      ┌──────────┐
│  高德地图3D  │ ◄─── fetch() ───► │  API Server   │ ◄──────────► │  SQLite  │
│  (浏览器端)   │   localhost:8503   │  (端口 8503)  │              │  DB      │
└──────────────┘                    └──────────────┘              └──────────┘
       ▲                                   ▲
       │ st.components.v1.html()          │ threading(daemon)
       │                                   │
┌──────┴──────────────────────────────────┴──────┐
│              Streamlit (端口 8501)               │
│  main() → load_base_stations() → build_map_html │
└─────────────────────────────────────────────────┘
```

- **Streamlit**：加载基站数据，生成完整 HTML 页面，通过 `st.components.v1.html()` 嵌入
- **HTTP API Server**：在 Streamlit 启动时以 daemon 线程运行，处理前端 JS 的数据请求
- **高德地图3D**：在浏览器端渲染地图、扇形、散点、栅格等覆盖物，支持3D旋转和俯视

### 5.2 API 端点

| 端点 | 参数 | 返回 | 用途 |
|------|------|------|------|
| `/api/measurements` | gnbid, ci, metric(rsrp/sinr/coverage), indoor | `{points, count, avg, grades}` | 散点模式数据（coverage模式avg=覆盖率%） |
| `/api/grid_analysis` | gnbid, ci, indoor | `{squares[{..., weak_count, weak_pct}], grid_params, total_count, total_avg_rsrp, grid_count}` | 栅格模式数据（含覆盖比例） |
| `/api/grid_cell_detail` | origin_lng, origin_lat, cell_lng, cell_lat, gx, gy, freq, indoor | `[{nr_earfcn, gnbid, ci, avg_rsrp, avg_sinr, count}]` | 栅格详情 |
| `/api/point_detail` | gnbid, ci, lng, lat, indoor | `{gnbid, ci, nr_earfcn, nr_ssb_rsrp, ...}` | 采样点详情 |
| `/api/optimal_azimuth` | gnbid, ci, indoor | `{optimal_azimuth, optimal_ratio, current_azimuth, current_ratio, total_count}` | 波束最优方位角 |

### 5.3 前端 JS 逻辑

#### 数据流

```
用户点击扇形 → toggleSelect(name)
  ├── 散点模式 → loadScatter(name, metric)
  │     ├── metric=rsrp/sinr → fetch /api/measurements → 渲染 CircleMarker
  │     ├── metric=coverage → fetch /api/measurements → 按覆盖/未覆盖着色
  │     └── metric=beam → fetch /api/measurements + fetch /api/optimal_azimuth
  │           → 按波束内/外着色 + 显示最优方位角
  └── 栅格模式 → loadGrid(name)
        └── fetch /api/grid_analysis → 渲染 GridPolygon
              ├── gridMode=cell → pctColor着色（小区占比）
              └── gridMode=weak → weakColor着色（覆盖比例）

悬停 2 秒 → startShowTimer(type, obj)
  ├── grid → fetchGridDetail(gp) → fetch /api/grid_cell_detail → InfoWindow
  └── scatter → fetchScatterDetail(marker) → fetch /api/point_detail → InfoWindow
```

#### 模式切换

- 散点/栅格互斥：切换时隐藏对方覆盖物，显示己方覆盖物
- 散点子模式：RSRP/SINR/波束/覆盖率 四选一，切换时重新加载所有已选小区
- 栅格子模式：小区占比/覆盖比例 二选一，切换时仅重绘颜色和统计（不重新请求数据）
- 室内/室外：切换时重新加载所有已选小区的当前模式数据

#### 统计面板

- **RSRP/SINR模式**：汇总采样点总数、加权均值、4级等级分布
- **波束模式**：波束内/外采样点数及占比 + 最优方位角及占比 + 当前方位角占比
- **覆盖率模式**：采样点总数、覆盖/未覆盖点数、覆盖率%
- **栅格-小区占比**：栅格数、小区采样点总数、总采样点数、RSRP均值、覆盖率等级分布
- **栅格-覆盖比例**：栅格数、覆盖面积（栅格数×25m²）、弱覆盖占比、覆盖比例等级分布

### 5.4 交互流程

1. **启动**：Streamlit 加载基站数据 → 生成 HTML → 内嵌 AMap 3D → 绘制所有扇形
2. **选小区**：点击扇形 → 高亮 → 加载散点数据 → 渲染 CircleMarker
3. **切模式**：散点/栅格/RSRP/SINR/波束/覆盖率 切换 → 隐藏旧覆盖物 → 加载新数据
4. **查详情**：悬停 2 秒 → 发起 API 请求 → 弹出 InfoWindow
5. **取消选**：点击已选扇形 → 递增序号 → 清除覆盖物
6. **搜小区**：输入关键字 → 过滤小区名 → 点击结果 → 自动选中+定位

## 6. 性能优化

### 6.1 覆盖索引优化

**问题**：25M 行 data 表的 longitude/latitude 为 TEXT 类型，`CAST(longitude AS REAL)` 导致 SQLite 无法使用索引，grid_analysis 的 PLMN 查询需全表扫描（54s）。

**方案**：
1. 新增 `lng REAL` / `lat REAL` / `rsrp REAL` / `sinr REAL` 列
2. 创建覆盖索引 `idx_data_plmn_earfcn_lng_lat_rsrp_sinr ON data(plmn, nr_earfcn, lng, lat, rsrp, sinr)`
3. SQL 查询改用数值列，PLMN 空间查询走覆盖索引
4. sinr 纳入覆盖索引避免回表，支持覆盖比例计算

**效果**：grid_analysis 从 58s 降至 2.2s（26x 提升）

### 6.2 最优方位角算法优化

**问题**：暴力扫描 O(72×N)，N=3000+ 时约 216000 次比较。

**方案**：区间覆盖法 — 对每个 bearing，将其波束影响区间 [b-30°, b+30°] 内的13个角度索引各+1，最后扫描72个角度取最大值。

**效果**：O(13×N)，提升约5倍。

### 6.3 query_measurements 列优化

**问题**：rsrp/sinr 模式查询 `SELECT lng, lat, rsrp, sinr`，但只需其中一列，sinr 不在覆盖索引中导致回表。

**方案**：改为 `SELECT lng, lat, {val_col}`，只查所需列。

**效果**：减少不必要回表和数据传输。

### 6.4 DB 连接合并

**方案**：grid_analysis 的 `query_cell_points` 和 `query_plmn_points_with_stats` 共享同一 DB 连接，从 3 次连接减为 2 次。

### 6.5 with_signal 条件现代化

**问题**：`build_where(with_signal=True)` 生成 `nr_ssb_rsrp!=""` 等TEXT列条件，无法使用索引。

**方案**：改为 `rsrp IS NOT NULL AND sinr IS NOT NULL`，使用REAL列。

### 6.6 bounds 顺序修复

**问题**：`poly.bounds` 返回 `(minx, miny, maxx, maxy)`，但 SQL 条件 `lng>=?, lng<=?, lat>=?, lat<=?` 需要 `(minx, maxx, miny, maxy)` 顺序。

**修复**：在 `query_grid_detail_points` 中将 bounds 重排为 `(minx, maxx, miny, maxy)`。

### 6.7 优化前后性能对比

|-----|--------|--------|------|
| measurements | ~6s | 1.9s | 3.2x |
| grid_analysis | 58s | 2.2s | 26x |
| grid_cell_detail | bug(返回空) | 0.04s | 修复 |
