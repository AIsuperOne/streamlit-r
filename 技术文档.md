# 5G基站扇形覆盖图 — 技术文档

## 1. 项目概述

基于 Streamlit + 高德地图3D 的5G基站覆盖可视化与智能分析工具。咸宁地区 2595万行MR采样 + 5.3万基站小区数据。

### 技术栈

| 层 | 技术 |
|---|---|
| 后端 | Python 3 + SQLite + Shapely |
| 前端 | Streamlit + 高德地图 JS API 2.0（3D视角） |
| 通信 | Streamlit 内嵌 HTTP API（端口 8503） |
| 智能体 | GLM-5.1 function calling + 2工具 + CLAUDE.md |

### 代码架构

```
app.py (~2185行)
├── 算法层     ~40行   reconstruct_polygon, build_where
├── 数据层     ~130行  query_measurements, query_grid_detail_points,
│                      query_point_detail, load_base_stations, export_cell_csv
├── API层      ~60行   MeasureAPI + start_api_server
├── 评估层     ~30行   evaluate_cell (扁平适配JS前端)
└── 呈现层     ~1840行 HTML/JS模板 + Streamlit入口

agent/
├── models.py      数据结构：GridSquare, CellAnalysis, 频段常量, 频点基准
├── algorithm.py   纯算法：方位角/距离/栅格构建/波束分析（无DB依赖）
├── evaluation.py  评估标准：合格线常量/RF专项计算/评估判定
├── analysis.py    核心编排：analyze_cell(单连接→7维+5射频全量), compute_grid_data
├── agent.py       智能体主类：对话循环 + tool_use调度

    └── file_read.py         CSV读取（限100行）
```

### 性能架构

- 单连接查询：`analyze_cell()` 单次DB连接贯穿全部查询，RF分布从已查询数据内存计算
- 数学落格：栅格归属判定用 `gx=floor((lng-origin)/cell_size)` 直接映射
- 缓存复用：`compute_grid_data()` 和 `query_optimal_azimuth()` 复用 `analyze_cell` 缓存，无额外查询
- 层次化数据：Agent通过1次 `query_cell_analysis` tool_call获取 scale/structure/quality/rf/stats 层次化结构

## 2. 功能说明

### 2.1 基站扇形覆盖图

加载所有有效基站，绘制扇形覆盖区域：120°波束（方位角±60°），默认50m半径（可选100/150/自定义1-5000m），按频段着色（700M金色/2.6G蓝色/4.9G红色），3D俯视pitch=50°，点击选中高亮绿色。

**频段筛选**：小区搜索上方3个切换按钮，控制地图上不同频段扇形显示/隐藏：

| 按钮 | 频段 | 频点 |
|---|---|---|
| 700M | 700M | 152650 |
| 2.6G | 2.6G | 504990, 524910, 529230 |
| 4.9G | 4.9G | 721824 |

隐藏频段时自动取消该频段已选小区，搜索结果同步过滤。

### 2.2 散点测量模式

| 子模式 | 着色 | 统计 |
|---|---|---|
| 覆盖率 | RSRP>-105且SINR>-3绿/否则红 | 覆盖/未覆盖数、覆盖率% |
| RSRP | 4级：优≥-95绿/中-105~-95黄/差-115~-105橙/极差<-115红 | 点数、加权均值、4级分布 |
| SINR | 4级：优≥15绿/中5~15黄/差-3~5橙/极差<-3红 | 点数、加权均值、4级分布 |
| 波束 | ±60°内绿/外红 | 波束内/外数及占比、最优方位角（散点）及占比 |
| 主邻差 | 4级：强干扰<-3dB红/中干扰-3~3dB橙/轻干扰3~6dB黄/无干扰>6dB绿 | 干扰点数、干扰比例、平均主邻差 |

### 2.3 栅格分析模式

5m×5m正则栅格，三个子模式：

| 子模式 | 着色指标 | 等级 |
|---|---|---|
| 小区占比 | pct=小区点/总点×100 | 非主服0~20%/弱主服20~50%/竞争50~80%/主控80~100% |
| 覆盖比例 | weak_pct=弱覆盖点/总点×100 | 优良0~20%/良好20~50%/稍弱50~80%/很弱80~100% |
| 波束覆盖 | 栅格中心方位角±60°判定 | 波束内绿/外红，7项统计指标 |
| 小区覆盖 | 每栅格采样点最多的小区颜色 | 15色色板+灰色(其他) |

覆盖比例模式统计：栅格数、覆盖面积、弱覆盖占比、栅格优良率。等级统计排除plmn_count=0的无效栅格。

波束覆盖统计（7项）：栅格数、波束内栅格、波束内占比（栅格）、主服波束占比（栅格）、最优方位角（栅格）、最优波束占比（栅格）、最优主服波束占比（栅格）。最优方位角采用pct加权法，与评估一致。多小区时独立计算后累加。

小区覆盖模式：每个栅格统计同频各小区采样点数，用占比最多的小区对应颜色渲染栅格。Top 15小区独立着色，占比<1%的小区不显示，其余归为"其他"(灰色)。扇形覆盖图颜色与栅格颜色同步。

### 2.4 详情弹窗

悬停2秒弹出InfoWindow：散点显示GNBID/CI/频点/RSRP/SINR/邻区信息；栅格显示采样点数/覆盖率/按小区分组统计。

### 2.5 AI小区评估

点击"AI评估"→ 导出CSV → 逐小区7维+5射频评估 → 显示评估卡片+AI分析文本。

## 3. 7维评估框架 + 射频5专项 + 综合评估

### 7维评估

| 层次 | 维度 | 指标 | 合格线 |
|---|---|---|---|
| 规模 | 业务水平 | 采样点数 / 同频点均值 | ≥1.0 |
| 规模 | 面积水平 | 栅格面积 / 同频点均值 | ≥1.0 |
| 结构 | 正对用户 | 最优方位角（散点）/ 波束内占比（散点）/ 最优波束占比（散点） | ≥60% |
| 结构 | 正对栅格 | 最优方位角（栅格）/ 波束占比（栅格）/ 最优波束占比（栅格） | ≥80% |
| 结构 | 主服覆盖 | 最优方位角（栅格）/ 主服波束占比（栅格）/ 最优主服波束占比（栅格） | ≥80% |
| 质量 | 散点覆盖率 | RSRP>-105且SINR>-3占比 | ≥95% |
| 质量 | 栅格优良率 | 覆盖优良(0~20%)的有效栅格占比(plmn>0) | ≥95% |

### 射频5专项

| 专项 | 指标 | 合格线 |
|---|---|---|
| 覆盖不足 | 弱覆盖比例/极弱占比/弱覆盖方位分布 | ≤5% |
| 越区覆盖 | 远距采样占比/最大距离 | ≤10% |
| 精准覆盖 | 波束内占比/方位角偏差/最优方位角 | ≤30°(偏差) |
| 重叠覆盖 | 同频干扰比例/平均主邻差/干扰方位分布 | ≤10% |
| 背向覆盖 | 背向覆盖占比（正对反方向±60°内采样点占比） | ≤10% |

### 综合评估

评分规则（满分100分）：

| 评分项 | 满分 | 计算方式 |
|---|---|---|
| 规模评分 | 50 | min(50, 50 × 业务量比值 × 面积覆盖比值)，乘积≥1时满分 |
| 质量评分 | 30 | min(30, 30 × (覆盖率/95) × (优良率/95))，乘积≥1时满分 |
| 射频评分 | 20 | 5项各4分，达标得4分不达标0分（覆盖不足/越区/精准/重叠/背向） |

判定规则：

| 条件 | 结论 |
|---|---|
| 7维全合格 + 射频5项全合格 | 综合合格 |
| 业务量或覆盖面积不合格 | 增加小区覆盖和对准用户 |
| 射频任一项不合格 | 射频待优化，需调整 |

## 4. 核心算法

### 4.1 栅格构建管线

`analysis._build_grid_squares()` 统一构建含center/sw/ne/bearing的GridSquare：

1. **原点确定**：`random.seed(gnbid_ci)` 选参考点，偏移半格 → `(origin_lng, origin_lat)`
2. **数学落格**：`gx = floor((lng-origin)/cell_size)`，统计每格cell_count
3. **PLMN统计**：数学落格O(1)定位候选栅格，直接通过gx/gy键映射判定归属 → 统计plmn_count和weak_count
4. **GridSquare构建**：计算center、sw/ne坐标、从基站到中心的方位角bearing

一次调用完成全部栅格属性，供可视化、评估、波束分析共用。

### 4.2 最优方位角

两种算法均用区间覆盖法（5°步长扫描72个角度），区别在权重：

**散点法**（维度3，最优方位角（散点））：每个采样点权重=1，找波束内点数最多的方位角，直接反映"天线是否对准用户"。

**栅格加权法**（维度4 + 前端波束覆盖模式，最优方位角（栅格））：每个栅格权重=pct/100，找波束内pct加权最大的方位角，反映"哪个方向小区主导性最强"。前端JS与Python使用相同算法，保证评估与显示数值一致。

### 4.3 波束覆盖

`analysis._grid_beam_coverage()` 计算当前/最优方位角下pct≥50%栅格占比：

- **当前波束覆盖**：当前方位角±60°内，pct≥50%栅格 / 波束内栅格数
- **最优主服波束占比（栅格）**：栅格加权法最优方位角±60°内，pct≥50%栅格 / 波束内栅格数

### 4.4 方位角计算

Haversine公式：正北0°，顺时针0~360°。Python端`calc_bearing`与JS端`calcBearing`逻辑一致。

### 4.5 单连接查询架构

`analyze_cell()` 采用单连接模式：打开一个SQLite连接，贯穿基站信息→散点统计→散点方位角+rsrp→干扰+lng/lat→栅格构建→PLMN区域统计全部查询，最后关闭。所有RF分布（弱覆盖方位、干扰方位、距离分布、背向比例）从已查询数据内存计算。

## 5. 数据层

### 5.1 数据库结构

- **数据库**：`xianning.db`（SQLite），25,955,709行MR采样 + 53,316条基站小区

### 5.2 data表 — 使用字段

| 字段 | 类型 | 用途 |
|---|---|---|
| gnbid | TEXT | 基站标识 |
| ci | TEXT | 小区标识 |
| plmn | TEXT | PLMN标识（46000=中国移动） |
| nr_earfcn | TEXT | NR频点号 |
| lng/lat | REAL | 经纬度 |
| rsrp/sinr | REAL | 信号强度/信噪比 |
| in_out_door | TEXT | 室内/室外标识 |
| nr_neighbor_* | TEXT | 邻区RSRP/PCI/频点列表 |

lng/lat/rsrp/sinr为REAL类型，避免TEXT列CAST导致全表扫描

### 5.3 5GBaseStation表 — 使用字段

小区中文名、物理站名、Gnbid、Cellid、nRPCI、RRU经度/纬度、方位角、天线挂高、工作频段、频点、覆盖类型

### 5.4 索引

| 索引 | 用途 |
|---|---|
| idx_data_gnbid_ci | 小区维度查询 |
| idx_data_gnbid_ci_plmn | 小区+PLMN查询 |
| idx_data_plmn_earfcn_lng_lat_rsrp_sinr | PLMN空间查询覆盖索引 |
| idx_data_plmn_earfcn_lng_lat_gnbid_ci | PLMN空间查询+小区覆盖索引（小区覆盖渲染） |
| idx_bs_gnbid_cellid | 基站查询 |

## 7. 智能体架构

```
┌─────────────┐   HTTP POST    ┌──────────────┐
│  app.py     │ ──────────────► │  GLM-5.1 API │
│ evaluate_   │   (带tools参数)  │  (智谱AI)     │
│ cell()      │ ◄────────────── │              │
└─────────────┘   分析文本       └──────┬───────┘
       ▲                                │ tool_calls
       │ 前端卡片展示                     ▼
                                ┌──────────────────┐
                                │  _TOOL_DISPATCH   │
                                │  ├── query_cell_   │
                                │  │   analysis      │
                                │  │   → 7维+5射频   │
                                │  │   层次化数据     │
                                │  ├── read_csv      │
                                │  └── query_sql     │
                                └──────────────────┘
```

**2个工具**：

| 工具 | 类型 | 功能 | 参数 |
|---|---|---|---|
| query_cell_analysis | Skill | 7维+5射频完整评估数据（层次化） | gnbid, ci, indoor |
| read_csv | MCP | 读取output/下CSV | filename |
| query_sql | MCP | 执行SELECT查询 | sql |

**对话循环**：加载CLAUDE.md → 构造user消息 → 调用GLM-5.1 → 若tool_calls则执行+回传 → 若content则提取分析文本。最大10轮。

**频点基准预计算**：`python3 -m agent.precompute_baseline` 预计算各频点的avg_samples和avg_grids，输出到 `output/freq_baseline_indoor{0/1/2}.json`。
